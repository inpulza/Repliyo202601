DIAGNÓSTICO COMPLETO: Problema de Etiquetas Visuales
RESUMEN DEL PROBLEMA
Los mensajes enviados desde Repliyo (manualmente o con IA) no muestran la etiqueta visual correcta ("Enviado desde Repliyo" o "Respondido con IA") en algunos casos, principalmente en DMs.

CAUSA RAÍZ DEL PROBLEMA
Los mensajes fueron enviados desde Repliyo ANTES de implementar la protección completa. Cuando Metricool sincronizó esos mensajes de vuelta:
El mensaje original tenía source='repliyo_auto' y author='Inpulza Testing'
Metricool devolvió el mismo mensaje con source='metricool_sync' y author='squareonpix'
La reconciliación sobrescribió el source y author porque la protección no estaba implementada correctamente
EL FRONTEND ESTÁ CORRECTO
El código de visualización funciona perfectamente:
// Líneas 892-893 de Inbox.tsx

const isSentByAI = isAutoReply(msg.source);              // ✅ Funciona

// Líneas 957-968: Muestra la etiqueta si source es correcto
{isSentFromRepliyo && (
  <span>{isSentByAI ? "Respondido con IA" : "Enviado desde Repliyo"}</span>
)}

El problema NO está en el frontend. El frontend lee el campo source correctamente.
EL PROBLEMA REAL: DATOS HISTÓRICOS CORRUPTOS
Los mensajes enviados ANTES de la corrección tienen:
source = 'metricool_sync' (incorrecto, debería ser 'repliyo_auto')
author = 'squareonpix' (incorrecto, debería ser el nombre de la marca)
¿Por qué? Porque cuando Metricool reconcilió esos mensajes, no había protección implementada y sobrescribió los campos.
SOLUCIONES POSIBLES
Opción 1: Migración de datos históricos
-- Identificar y corregir mensajes históricos

    author = 'Inpulza Testing',
    direction = 'outbound'
WHERE content LIKE '%¡Hola @inpulza!%' 
  AND source = 'metricool_sync'
  AND direction = 'outbound';
