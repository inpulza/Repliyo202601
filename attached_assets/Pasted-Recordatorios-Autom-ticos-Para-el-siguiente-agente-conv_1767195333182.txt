Recordatorios Automáticos
Para el siguiente agente/conversación:

El proyecto cuenta con un Sistema de Recordatorios Automáticos (Smart Customer Follow-up System) completamente implementado que envía seguimientos personalizados a clientes inactivos tanto en DMs como en comentarios. La arquitectura incluye dos tablas principales en PostgreSQL: reminder_rules (configuración por marca con campos como enabled, delayHours1, delayHours2, maxReminders, dailyBrandCap, useAiContent, templateText, applyToDms, applyToComments, autoCloseAfterMaxReminders, autoCloseDelayHours) y reminder_events (registro de cada recordatorio con estado scheduled, sent, failed, cancelled, campos content, scheduledAt, sentAt, reminderNumber, errorMessage, etc., más un índice parcial único para prevenir duplicados). El storage layer (server/storage.ts) expone 14+ métodos CRUD incluyendo getReminderRules(), upsertReminderRules(), scheduleReminderAtomic() (transacción atómica con precheck integrado), checkConversationEligibleForReminder() (validación de elegibilidad), countDailyReminders() para enforcement del cap diario, y métodos de analytics como getReminderStats(), getReminderTimeline(), getReminderFailureReasons() con soporte para rangos temporales ('today', '7d', '30d'). El ReminderService (server/services/reminderService.ts) implementa un flujo de 3 capas defensivas: primero ejecuta checkConversationEligibleForReminder() para validar elegibilidad antes de llamar a la IA (ahorrando cuota), luego genera contenido personalizado via LLM si useAiContent=true, y finalmente programa atómicamente con scheduleReminderAtomic(). Las APIs REST están en server/routes.ts bajo /api/brands/:brandId/reminder-rules (GET/PUT), /api/brands/:brandId/reminder-events (GET), /api/brands/:brandId/reminders/run (POST para ejecución manual), y endpoints de analytics /api/brands/:brandId/reminders/analytics/summary|timeline|failures?timeRange=. El frontend incluye ReminderSettingsForm.tsx con secciones para Canales de Aplicación, Tiempos de Espera, Límites, Cierre Automático, Contenido del Mensaje, dashboard de métricas (KPIs, progress bar de cuota diaria, resumen de estados, razones de fallo), e historial de eventos. Los hooks de React Query están en client/src/hooks/useReminderRules.ts (useReminderRules, useBrandReminderEvents, useReminderStats, useReminderTimeline, useReminderFailures).
