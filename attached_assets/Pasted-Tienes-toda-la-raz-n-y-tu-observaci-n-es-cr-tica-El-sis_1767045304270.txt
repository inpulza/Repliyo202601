🔍 El Diagnóstico: ¿Por qué no se guardaron los teléfonos?
Actualmente, el sistema funciona de forma "Reactiva":

Llega el mensaje (DM).

El sistema evalúa: "¿Debo responder automáticamente?".

Solo si la respuesta es SÍ, la IA lee el mensaje y ejecuta la función update_contact.

Si la respuesta es NO (porque estás en modo manual, o cooldown, o el mensaje es corto), el sistema ignora el contenido. El dato se pierde.

El fallo: Es como tener una secretaria que solo anota el teléfono del cliente si decide hablar con él. Si decide quedarse callada, tira la nota a la basura.

🛠️ La Solución Arquitectónica: "El Servicio de Enriquecimiento Pasivo"
Para replicar la robustez de un CRM real (como Respond.io, que usa workflows para capturar datos independientemente de la respuesta ), debemos separar la "Extracción" de la "Respuesta".

Implementaremos el Contact Enrichment Service (Servicio de Enriquecimiento). Este será un "sabueso silencioso" que huele todos los mensajes entrantes.

Cómo funcionará el nuevo flujo:
Entrada: Llega un DM (WhatsApp, Instagram, etc.).

Paso Paralelo 1 (El Sabueso): El Enrichment Service escanea el texto inmediatamente.

Capa 1 (Regex/Patrones): Busca patrones obvios como correos (@) o teléfonos (+52..., 600...) instantáneamente.

Capa 2 (IA Extractora): Si detecta intención de datos (ej: "Me llamo Juan"), usa una IA pequeña y rápida (sólo para extracción, no para conversar) para sacar el JSON de datos.

Acción: Actualiza el CRM (crm_contacts) en tiempo real.

Paso Paralelo 2 (El Agente): El sistema decide si responder o no (Auto-reply), pero el dato ya está guardado.
