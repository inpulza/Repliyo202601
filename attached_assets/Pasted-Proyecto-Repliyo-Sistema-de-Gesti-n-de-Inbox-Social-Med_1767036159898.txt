El sistema incluye respuestas automáticas con IA usando OpenAI o Gemini, configurables por agente.

Estado actual y lo que estamos validando:

En la conversación anterior trabajamos en corregir varios problemas críticos del sistema de contexto y resúmenes. Los cambios ya están implementados pero necesitamos verificar que funcionan correctamente. Los problemas que resolvimos (en teoría) son:

SummaryService no distinguía DMs de comentarios: El servicio usaba "comment" como messageType hardcodeado para todos los casos, lo que causaba que el filtrado de historial (filterHistoryByAuthor) funcionara incorrectamente para DMs. Corregimos esto para que ahora obtenga conversation.type de la BD (dm o comment) y pase el messageType correcto: 'conversation' para DMs (incluye todo el historial) y 'comment' para comentarios (filtra por autor).

Resúmenes truncados/corruptos: Encontramos que los resúmenes en la tabla conversation_user_summaries se guardaban truncados (ej: solo 67 caracteres cuando deberían tener 300+ palabras). Agregamos validación de longitud mínima (100 caracteres) para que resúmenes incompletos no se persistan. También eliminamos los resúmenes corruptos existentes de jordan.inpulza.

createDummyMessage con type hardcodeado: La función creaba mensajes dummy con type: "comment" siempre, afectando el filtrado downstream. Ahora acepta el tipo de conversación real como parámetro.

SummaryService ya usa el LLM del agente: Confirmamos que ahora usa createLLMProvider(agent, secrets) con generateRawCompletion, respetando la configuración de OpenAI/Gemini del agente en lugar de tener Gemini hardcodeado.
